# 15分钟动态选股数据字段标准

> 目的：自动化每15分钟只在数据足够时输出真实候选。  
> 缺关键字段时，必须显示“数据不足”，不能编价格、评分或止盈止损。

## 1. 必需行情字段

每只股票必须有：

- `current_price`：当前价。
- `previous_close`：上一交易日收盘价。
- `open_price`：今日开盘价。
- `high_price`：当日最高价。
- `low_price`：当日最低价。
- `change_pct`：当日涨跌幅。
- `volume`：成交量。
- `turnover`：成交额，美股可用成交量和美元成交额。
- `timestamp`：行情时间戳。
- `benchmark_change_pct`：对应基准涨跌幅。
- `sector_change_pct`：行业或主题板块涨跌幅。

缺任一关键价格字段，不给真实买入区/止盈/止损。

## 2. 必需风险字段

A股：

- 是否停牌。
- 是否涨停/跌停或接近涨跌停。
- 是否临近财报/重大公告/监管问询。
- 是否处于集合竞价、午休或尾盘阶段。
- 是否 T+1 影响卖出计划。

美股：

- 是否临近财报。
- 是否有盘前/盘后重大新闻。
- 盘前/盘后价格变化。
- VIX 或市场风险偏好。
- 对应 ETF：SPY、QQQ、SMH 的相对表现。

## 3. 派生评分字段

每轮自动计算：

- `trend_score`：趋势强度，0-20。
- `volume_price_score`：量价配合，0-15。
- `relative_strength_score`：相对强弱，0-20。
- `sector_sync_score`：板块共振，0-15。
- `event_risk_score`：事件风险，0-10，风险越低分越高。
- `overnight_fit_score`：隔夜适配，0-20。
- `total_score`：总分，0-100。

## 4. 进入动态推荐的最低条件

- 总分不低于 70。
- 当前价和时间戳可靠。
- 不是停牌。
- 未处于无法正常交易状态。
- 有明确买入观察区、止盈、止损和卖出/放弃条件。
- 隔夜候选必须有隔夜评分和隔夜风险说明。

## 5. 买入区/止盈/止损生成原则

买入观察区：

- 不能直接用一个固定价格，必须是区间。
- 由当前价、分时支撑、开盘价、短周期强弱和波动率决定。
- 如果波动过大，买入区必须扩大或标为“不适合追入”。

止盈区：

- 至少考虑当日波动、前高、整数关口、板块强度。
- 高波动股止盈区要更保守。

止损价：

- A股必须考虑 T+1，不能假设当天可卖出。
- 美股可用开盘低点、短周期支撑或波动率止损。

卖出/放弃条件：

- 必须是可观察条件，例如跌破开盘低点、弱于基准、板块转弱、放量长上影、盘后利空。

## 6. 输出状态

每轮只有三种状态：

- `真实推荐`：数据完整，评分达标。
- `观察不足`：有部分数据，但分数或风险条件不达标。
- `数据不足`：缺关键行情或风险字段。
